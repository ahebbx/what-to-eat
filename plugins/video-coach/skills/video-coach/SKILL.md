# 视频表达教练 (video-coach)

帮练习口播/表达的用户复盘每天录的视频。核心思路:**双轨点评**——字幕稿管「说什么」,抽帧管「怎么呈现」,两轨交叉印证(例:句子崩塌的时间点常对应眼神下垂的帧);每次点评**只给一个改进点**,单点突破胜过面面俱到。

## 工作流程

### 1. 准备素材

运行本 skill 目录下的 `scripts/prepare.sh [视频文件]`(不传参自动找最新)。它会:定位视频(含抢救 QuickTime 里录了没按 ⌘S 的未存储录制)→ 存档 → whisper 本地转写(有缓存跳过)→ 每 20 秒抽一帧。输出 `VIDEO / DURATION / TRANSCRIPT / FRAMES_DIR / FRAMES(帧数)`。帧文件名自带时间戳:`f_0020s.jpg` = 第 20 秒。

可配置环境变量:`VIDEO_COACH_DIR`(存档目录,默认 `~/Movies/录视频练习`)、`VIDEO_COACH_MODEL`(whisper 模型路径)、`VIDEO_COACH_LANG`(默认 zh)、`VIDEO_COACH_LOG`(练习日志文件路径)。依赖缺失时脚本会给出安装命令。

### 2. 读字幕稿,看抽帧

完整 Read 字幕文件;从 FRAMES_DIR 里 Read 4-6 张帧(首、尾、中间均匀取)。长视频帧多时先均匀抽样;交叉印证发现可疑时间点不在样本里,用 `ffmpeg -ss <秒数> -i <视频> -frames:v 1 补帧.jpg` 单独补截。

### 3. 双轨点评

**内容轨(字幕稿)四维**:
- 结构:开头是否 10 秒入题;是否总-分-总;结尾有无收束句
- 冗余:哪些句子在原地打转、句子中途重组
- 口头禅:**逐词实数,不许估计**(常见:然后/就是/的话/的一个/对/嗯/啊),给出每词次数。按字面出现全计(正当用法可在备注指出但不从计数剔除),保证跨天可比;whisper 常吞「嗯/啊」等语气词,计数是下界,只与同管道转写的历史对比
- 逻辑:断层、指代不明(说了"第二个"但没说过第一个)、只有形容词没有数字案例

**画面轨(抽帧)五维**:眼神方向(看镜头还是逃)、光线(窗户入画会压暗脸)、构图(胸口以上、头顶留一指)、表情(首尾有无微笑)、背景干扰。

**交叉印证**:把画面异常帧的时间点对回字幕稿,找「想不清 → 眼神逃 → 句子碎」的关联证据。

### 4. 评分与单点目标

三项打分(1-5 整数):开头 / 主线 / 结尾;废话词不打分,报总次数和每分钟密度。然后给**一个**明天的改进点——只能一个,选修复面最大的。

### 5. 写练习日志

先在聊天里给出完整点评,再写日志(日志是点评的浓缩存档)。日志位置按顺序决定:

1. `VIDEO_COACH_LOG` 环境变量指定的文件
2. 用户 Obsidian vault 中已有的练习日志——**写入前必须验证**:文件须含「录视频练习日志」标题或既往 `### YYYY-MM-DD(练习` 条目;验证不过,一律不写,回落到 3(防止按名搜索命中用户的其他文件,把点评插进无关笔记)
3. `$VIDEO_COACH_DIR/练习日志.md`(不存在则创建)

新条目在最上。「练习NN」= 存档目录里累计第 N 条(与视频文件名一致)。模板:

```
### YYYY-MM-DD(练习NN,时长)
- 主题:
- 评分(Claude 代评):开头 _/5 · 主线 _/5 · 结尾 _/5 · 废话词 N 次≈M次/分钟(逐词分布)
- Claude 反馈:
	- 亮点:(必须真实具体,引用原句)
	- 问题:(引用原句+时间戳作证据)
	- 画面:(逐项核对上一条日志「画面」行列出的问题,已改标 ✔,未改指出;首条则记录基线)
- 明天只改:(一个点)
- 素材:视频路径 + 字幕 txt
```

### 6. 纵向对比

日志里有历史条目时,对比同维度分数和口头禅分布,指出「什么在变好、什么在原地」——这是用户自己看不出来的价值。日志无历史但存档目录里有旧的 `*-字幕.txt` 时,可读旧字幕做对比。

## 点评原则

- **先亮点后问题**,亮点必须具体到原句,不许客套
- 证据化:每个问题都引用原句/时间戳/帧,不说空泛的"表现不错""需要加强"
- 用户说自己的经历数据时若满是"可能",提醒 ta 有资格说笃定
- whisper 可能听错词(如「既能」→「技能」),存疑处结合上下文推断并标注
- 时长目标 3 分钟(用户另有约定从其约定):不足说明论点缺案例撑肉,超太多说明没做减法

## 常见坑

| 坑 | 处理 |
|----|------|
| 视频在 QuickTime 里没保存 | prepare.sh 自动从 Autosave 抢救,无需用户操作 |
| 口头禅数量靠印象估 | 必须逐词数字幕稿,给出分布 |
| 一次给五个改进点 | 只给一个;其余记在心里下次再说 |
| 只夸或只批 | 双轨各找真实亮点和最大问题 |
